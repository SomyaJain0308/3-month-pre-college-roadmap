## Day 1 — Mock Interview #1

- [ ] Go to [Pramp.com](https://pramp.com) — free peer mock interviews
- [ ] Schedule and complete one mock interview session
- [ ] LeetCode timed session:
  - [ ] 3 problems, 25 mins each, zero hints, zero looking up
  - [ ] Explain your approach out loud as you code (practice this)
- [ ] After session — honest self-review:
  - [ ] What did I solve easily?
  - [ ] Where did I get stuck?
  - [ ] What patterns do I need to review?
- [ ] LeetCode: 3 more problems (targeted at your weak spots)
- [ ] Commit and push

## Day 3 — System Design Introduction

- [ ] Watch Gaurav Sen's system design playlist (first 3 videos)
- [ ] Understand these concepts at a high level:
  - [ ] Horizontal vs vertical scaling
  - [ ] Load balancer — what it does
  - [ ] Caching — why Redis exists
  - [ ] Database: SQL vs NoSQL — when to use each
  - [ ] CDN — what it is
  - [ ] Message queues — Kafka concept
- [ ] You won't be asked system design in Year 1 interviews — but planting this mental model now means it compounds
- [ ] LeetCode: 3 problems
- [ ] Commit and push
